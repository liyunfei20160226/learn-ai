## Codebase Patterns
- Vanilla JavaScript + HTML5 Canvas for whiteboard rendering
- Use CSS grid background for infinite canvas grid pattern
- Canvas transforms handled via context translate/scale with separate offset/scale state
- Zoom scales towards mouse position for intuitive interaction
- Toolbar is fixed on left with SVG icons for each tool
- Tool selection stored in Whiteboard class with simple string identifiers
- Shapes are stored as plain objects in elements array with type-specific properties
- Drawing workflow: mousedown starts shape, mousemove updates preview, mouseup completes and adds to elements
- Convert between screen coordinates and world coordinates using screenToWorld/worldToWorld methods
- Hold Shift to constrain drag-to-create shapes to uniform aspect ratio (perfect circle/square)
- Hold Shift constrains lines to 0°, 45°, or 90° angles for straight lines
- Arrowheads drawn using trigonometry to calculate angle-based vertices
- Freehand pencil stores points in an array, uses lineCap: 'round' and lineJoin: 'round' for natural smooth lines
- Text tool uses an HTML textarea overlay positioned over canvas for live editing
- After editing completes, text is drawn directly on canvas with fillText
- Diamond is a four-point polygon shape centered at the midpoint of the bounds
- Hit detection uses shape-specific algorithms: area-based for closed shapes, distance-based for lines
- Visual feedback during interactive tools like eraser improves UX significantly
- Selection stores references to selected element objects in selectedElements array
- Eight resize handles (corners and middles) for single element resizing
- Shift key maintains aspect ratio during resizing
- Bounding box calculation is element-type-specific for accurate selection
- Copy/paste uses clipboard JSON format to store element data for deep copying
- Undo/redo uses JSON serialization to take full snapshots of element array for history
- Save history only after interaction completes (not during drag/resize) for efficiency
- Style panel fixed on right with native HTML color inputs and range sliders for easy styling
- Current styles stored in this.styles object and used for all new elements drawn
- Persistence uses localStorage with JSON serialization - saves full elements, viewport, and styles
- Action buttons (undo/redo/clear/export) handled in selectTool() without changing current tool
- drawElement accepts optional target context parameter for rendering to off-screen canvas for export
- WebRTC collaboration uses simple-peer library for P2P connections with external signaling server
- Collaborative actions broadcast after interaction completes (on mouseup) for minimal network traffic
- Remote cursors displayed as colored dots with username labels for visibility
