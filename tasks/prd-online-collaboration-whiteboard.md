# PRD: 在线协作白板 MVP 版本

## 介绍

创建一个类似 Excalidraw 的在线协作白板，支持团队多人实时协作绘图。MVP 版本专注于核心绘图功能和基础实时协作，数据存储在浏览器本地。该产品解决团队远程协作时需要一起脑暴、绘制流程图、草图的沟通问题。

## 目标

- 提供流畅的白板绘图体验，支持常见基础图形
- 实现多人同时编辑的实时协作功能
- 支持看到其他协作者的光标位置
- MVP 使用浏览器本地存储，无需后端数据库
- 代码架构可扩展，方便后续添加云端存储功能

## 功能需求

- FR-1：系统必须提供无限画布，支持平移和缩放
- FR-2：系统必须提供完整基础工具集：选择、矩形、圆形、菱形、直线、箭头、铅笔、文本、橡皮
- FR-3：用户可以选择绘制元素的颜色和线条粗细
- FR-4：用户可以选择、移动、缩放、删除已绘制的元素
- FR-5：系统必须支持撤销和重做操作
- FR-6：画布内容自动保存到浏览器 localStorage
- FR-7：页面刷新后自动恢复画布内容
- FR-8：支持导出整个画布为 PNG 图片下载
- FR-9：支持多人通过 WebRTC P2P 实时协作编辑
- FR-10：显示其他协作者的光标位置和用户名
- FR-11：用户可以创建房间，并通过链接/房间码邀请他人加入

## 非目标

- 不支持云端存储和持久化保存（MVP 只有 localStorage）
- 不支持版本历史和版本回退（除了基本撤销/重做）
- 不支持分层分组
- 不支持锁定元素
- 不支持评论和批注
- 不支持图片导入
- 不支持手绘形状识别
- 不支持幻灯片演示模式
- 不支持移动端触摸操作优化（MVP 只支持桌面鼠标）

## 设计考虑

- UI 参考 Excalidraw 的简洁风格
- 工具栏居左，样式面板居右或顶部
- 默认背景带浅色网格
- 协作光标使用不同颜色区分用户
- 操作延迟尽量低，实时感强

## 技术考虑

- 使用 Canvas API 绘制画布（性能更好）或 SVG 方案（选择方便）
- 使用 WebRTC 实现 P2P 实时通信，不需要后端信令服务器可使用公共信令或简化方案
- 操作使用 OT 或简单顺序广播（MVP 用简单广播即可）
- 画布数据使用 JSON 格式保存
- 纯前端项目，不需要后端服务（除了可选信令服务器）
- 响应式，适应不同屏幕尺寸

## 成功指标

- 可以在 MVP 上完成基本的团队脑暴流程图绘制
- 协作延迟低于 500ms，有实时感
- 基本操作（创建、移动、删除）流畅无卡顿
- 刷新页面内容不丢失
