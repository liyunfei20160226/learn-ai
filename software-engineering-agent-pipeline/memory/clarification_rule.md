---
name: Clarification Rule
description: When requirements or questions are unclear, ask user for clarification
type: feedback
---

Rule: **If the user's request is ambiguous, unclear, or you are uncertain about what is being asked, proactively ask clarifying questions before starting work.**

Why: Avoid making incorrect assumptions that lead to wasted effort doing the wrong thing. Always confirm understanding first.

How to apply:
- If requirements are vague or incomplete → ask specific clarifying questions
- If multiple interpretations are possible → list options and ask user to confirm
- If you don't understand what the user wants → ask them to rephrase or elaborate
- Do not proceed with implementation until you have clear confirmation
