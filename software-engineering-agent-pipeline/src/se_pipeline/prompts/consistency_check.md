# 一致性检查 - 对比设计文档和实际代码实现

## 任务

对比**设计文档**和**实际代码分析结果**，检查实现是否和设计一致。**只需要输出发现的不一致和遗漏，不需要赞美，不需要说"大部分一致"这种话。如果完全一致，可以输出空列表。**

## 检查要点

1. **功能需求一致性**：代码实现是否覆盖了设计文档要求的所有功能？有没有功能设计了但没实现？

2. **架构一致性**：实际代码的架构是否符合设计文档的架构设计？模块划分是否一致？

3. **数据库一致性**：实际推导出的数据库表结构是否符合设计文档的数据库设计？有没有表/列遗漏？关系是否一致？

4. **API一致性**：API接口设计是否符合设计文档的API契约？路径、方法、参数是否一致？

## 输出要求

你必须按照指定的 YAML 格式输出：
- `inconsistencies`: 不一致列表，每个不一致必须包含：
  - `check_id`: 唯一ID
  - `location`: 功能名称/模块名称
  - `description`: 具体描述哪里不一致，设计说什么，实际代码是什么
  - `severity`: 严重程度
    - `error`: 严重不一致，设计要求但完全没实现，必须修复
    - `warning`: 部分不一致，实现和设计有出入，建议修正
- `summary`: 整体总结**只总结不一致问题**
