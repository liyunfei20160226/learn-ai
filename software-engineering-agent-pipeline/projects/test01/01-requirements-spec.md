# 需求规格说明书 - My Todos - 极简待办事项

**项目ID**: test01
**生成时间**: 2026-04-06 11:18:29
**验证状态**: ✅ 通过

## 概述

一个基于 Next.js (App Router) 构建的个人极简待办事项网站。项目采用 LocalStorage 进行数据持久化，支持任务的增删改查、状态标记及分页展示。界面使用 Tailwind CSS 和 shadcn/ui 打造柔和圆角风格，专注于单用户本地高效管理，无后端依赖，无多端同步。

## 用户角色

### 个人用户
唯一的系统使用者，在本地浏览器中进行待办事项的创建、管理和查看，无需注册登录。

## 功能需求

🔴 **FR1 任务新增**
- 1. 输入框固定在列表顶部，页面加载后默认聚焦。2. 按 Enter 键新增，成功后清空输入框并保持聚焦。3. 自动去除首尾空格。4. 禁止提交空内容（提示'任务内容不能为空'，输入框变红抖动，下方显示文字，一旦输入立即恢复）。5. 禁止添加重复项（忽略大小写，提示'该任务已存在'，交互同空内容校验）。6. 新增成功后自动跳转至第 1 页。

🔴 **FR2 任务列表展示**
- 1. 按创建时间倒序排列（最新在上）。2. 已完成任务保持原位置，不沉底。3. 已完成任务视觉表现为：文字添加划除线 + 透明度降低。4. 每页固定显示 20 条。5. 无数据时显示'暂无待办事项'提示，且不显示分页器。6. 布局固定为桌面端，不做移动端自适应。

🔴 **FR3 任务编辑**
- 1. 点击任务文字进入原地编辑模式（Inline Edit），变为输入框。2. 移动端/无悬停设备：点击任务进入编辑态时同时显示删除按钮。3. 桌面端：悬停显示操作按钮。4. 失去焦点或点击其他任务时自动保存并退出编辑。5. 编辑时若内容与其他任务重复（忽略大小写），报错提示（同新增），保持编辑态，输入即消除错误。6. 编辑成功无视觉反馈。7. 不支持取消编辑（修改错误可再次修改）。

🔴 **FR4 任务删除**
- 1. 点击删除按钮后立即永久删除，无二次确认。2. 若在非第 1 页删除导致当前页为空，自动跳转至上一页。

🔴 **FR5 状态标记**
- 支持点击复选框标记任务为'已完成'或'未完成'，状态变更实时保存至 LocalStorage。

🟡 **FR6 分页功能**
- 1. 分页器始终可见（除非无数据）。2. 包含'上一页'、'下一页'按钮及折叠策略的页码数字（如 1 ... 5 6 7 ... 50）。3. 首尾页按钮置灰不可点，当前页码高亮。4. 点击页码或翻页按钮，页面滚动条自动回到列表顶部。5. 严格锁定每页 20 条，不提供切换选项。

🔴 **FR7 数据持久化与异常处理**
- 1. 所有操作（增删改查）实时同步保存至 LocalStorage。2. 多标签页间数据独立，不自动同步。3. 若 LocalStorage 禁用或已满，弹出 shadcn/ui 风格的模态对话框进行阻断式警告。

## 非功能需求

🔴 **NFR1 技术栈要求**
- 使用 Next.js (最新 App Router 架构), TypeScript, Tailwind CSS, shadcn/ui。项目结构需符合当前流行的 Next.js 标准（含 Prettier/Lint 配置）。

🟡 **NFR2 UI 设计风格**
- 采用带圆角的柔和风格，统一圆角半径为 sm (4px)。仅针对现代主流浏览器（Chrome/Edge/Firefox 最新版）优化。

🟢 **NFR3 性能与兼容性**
- 无需 PWA 支持，无需深色模式，无需统计信息。数据仅存于本地，换设备丢失。

🔴 **NFR4 交付物要求**
- 提供完整的搭建指南（含初始化命令、shadcn 安装步骤）及核心代码文件。

## 不在范围内（明确不做）

- 多用户注册与登录系统
- 后端服务器存储与多端数据同步
- 子任务、文件附件、团队协作功能
- 任务分类、截止日期、优先级标记
- 数据导出/导入或一键清空功能
- 统计信息面板与深色模式切换
- PWA (渐进式 Web 应用) 支持
- 移动端自适应布局（固定桌面端布局）
- 键盘快捷键支持
- 复杂的动画效果（仅需划除线和错误抖动）
- 旧版浏览器兼容性支持
